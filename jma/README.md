# JMA Weather Data (Reproducible Pipeline)

This folder stores canonical JMA hourly datasets and the station-agnostic merge script.

## Canonical outputs

Generated by `jma/merge_clean_jma.py` and committed to the repository:

```
jma/jma_mikuni_hourly_8.csv      → Mikuni (三国) AMeDAS       prec=57 block=1071  (Node A)
jma/jma_fukui_hourly_8.csv       → Tsuruga (敦賀) main stn    prec=57 block=47631 (Node B proxy + Node D)
jma/jma_katsuyama_hourly_8.csv   → Katsuyama (勝山) AMeDAS   prec=57 block=1226  (Node C)
```

> **⚠ NAMING WARNING:** `jma_fukui_hourly_8.csv` contains **Tsuruga (敦賀)** data,
> not Fukui City. The file was misnamed when first created. The preset was
> confirmed via JMA ETRN block lookup: prec_no=57, block_no=47631 → 敦賀（福井県）.
> The actual Fukui City station is block_no=47616. Use `--station fukuicity` to
> fetch Fukui City data (outputs to `jma/fukuicity_rawdata/`).
>
> No JMA station (AMeDAS or main observatory) exists for Obama (小浜) or Mihama
> (美浜) in the ETRN system. Tsuruga (~15 km from Rainbow Line) is the closest
> available station for Node D.

Current demand model uses Mikuni by default (`jma_mikuni_hourly_8.csv`).

## Schema

| Column | Description |
|---|---|
| `timestamp` | Hourly datetime (local JST) |
| `snow_depth_cm` | Snow depth (cm) |
| `snowfall_1h_cm` | 1-hour snowfall (cm) |
| `temp_c` | Temperature (°C) |
| `precip_1h_mm` | 1-hour precipitation (mm) |
| `sun_1h_h` | 1-hour sunshine duration (h) |
| `wind_speed_ms` | Wind speed (m/s) |
| `weather_type` | Weather symbol code |
| `humidity_pct` | Relative humidity (%) |

`src/run_analysis.py` (via `src/data_loader.py`) maps to legacy names internally:
`temp_c` → `temp`, `precip_1h_mm` → `precip`, `sun_1h_h` → `sun`, `wind_speed_ms` → `wind`

## Adding new months (scripted downloads)

Raw hourly downloads are generated by the fetch script and then merged into
the canonical outputs. No manual file edits are expected.

1. Fetch monthly hourly CSVs (per station):
   ```bash
   python jma/fetch_jma_monthly.py --station mikuni --year 2026 --month 1
   python jma/fetch_jma_monthly.py --station fukui --year 2026 --month 1
   python jma/fetch_jma_monthly.py --station katsuyama --year 2026 --month 1
   ```
   Outputs land in:
   - `jma/mikuni_rawdata/`
   - `jma/fukui_rawdata/`
   - `jma/katsuyama_rawdata/`
2. Merge into canonical outputs:
   ```bash
   python jma/merge_clean_jma.py
   ```
3. The merge script upserts by timestamp (new rows win on duplicates).
4. Commit the updated canonical CSVs so all collaborators have current data.

## Notes

- Raw encoding is CP932 (Shift-JIS family).
- Script auto-detects JMA column positions from header rows.
- Upsert logic: newly parsed rows win for duplicate timestamps.
- Raw CSV files (inside `*_rawdata/`) should stay local (do not commit).
